1) ویژگی های هاب => [
    :: این ویژگی در لایه فیزیکی مدل OSI کار میکند 
    :: پر این سیستم اگز قرار باشد اطلاعات از یک کامپیوتر به یک کامپیوتر دیگر فرستاده شود , به تمام کامپیوتر های متصل به هاب نیز فرستاده میشود
    :: در این سیستم اگر یک کامپیوتر یک اطلاعات را به یک کامپیوتر دیگر ارسال کند بقیه کامپیوتر ها باید منتظر پایان ارسال شوند تا بتوانند خودشان ارسال اطلاعات کنند
    :: امنیت بسیار پایینی دارد
    :: پهنای بند آن به صورت اشتراکی است
    :: اگر به صورت همزمان دو کامپیوتر در هاب ارسال اطلاعات انجام دهند اصطلاحا collision یا تصادف اتفاق می افتد و آن هاب به طور کامل از بین میرود
    :: collision domain -> به محدوده ای که با اختلال روبرو شده است گفته میشود
    :: استصال Hub a , Hub b -> باید از طریق یک سیم یکی از ورودی های هاب را به یکی از ورودی های هاب دیگر وصل کنیم
]

2) bridge چیست => [
    :: برای اتصال بین دو هاب از آن استفاده میشود
    :: اگر قرار باشد به عنوان مثال دو هاب را به هم وصل کنیم در حالت عادی باید از طریق یک سیم این کار را انجام دهیم اما از طریق بریج این اتصال راحت تر است
    :: یک بریج قرار میدهیم که دارای دو پورت است و هر کدام از هاب ها به یکی از پورت های بریج متصل میشود 
    :: به محض اینکه کامپیوتر ها در هاب هایی مختلف روشن میشوند آدرس مک آن ها در جدولی در ساختار بریج ذخیره میشود
    // به این جدول ایجاد شده مک تیبل گفته میشود
    :: بریج در لایه دوم شبکه کار میکند 
    :: تفاوت بریج با حالت قبل از آن این بود که اطلاعات یک کامپیوتر که قرار بود ارسال شود را فقط در همان هاب نگه میداشت و اجازه ارسال به هاب دیگر را نمیداد اما همچنان اطلاعات در همان هاب بین همه کامپیوتر ها ارسال میشد
]

3) switch چیست => [
    :: در واقع یک multi port bridge است 
    :: در واقع یک بریج است که دارای چندین پورت است
    :: در این حالت کامپیوتر ها متصل به سوییچ آدرس مک آن ها در یک مک تیبل ذخیره میشود  و اگر قرار باشد یک کامپیوتر به یک کامپیوتر دیگر اطلاعات ارسال کند فقط به همان کامپوتر ارسال میشود وبرای بقیه کامپوتر ها ارسال نمیشود
    :: در هاب پهنای باند اشتراکی است ولی در سوییچ پهنای باند اختصاصی است
    :: در این حالت اگر collision رخ دهد فقط همان کامپیوتر را درگیر میکند 
    :: برای اتصال دو سوییچ به هم یکی از پورت های سوییچ اول را به یکی از پورت های سوییچ دوم وصل میکنیم یا از طریق پورت uplink میتوان آن هارا به هم وصل کرد
    :: وقتی دو سوییچ با هم وصل میشوند آدرس های مک خود را با هم به اشتراک میگذارند
]

    :: fragment free -> ترکیب دو و سه است
    :: fixed port -> دارای پورت ثابت است و قابل تغییر نیست
    :: stackable 
    :: modular 
    :: chassis
