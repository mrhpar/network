:: مدل شبکه : یک مدل مفهمومی است که در واقع وجود خارجی ندارد و فقط برای درک عملکرد شبکه از آن استفاده میشود
:: OSI -> operating system interconnection 
:: TCP / IP -> Transmission control Protocol / internet Protocol

// تشریح مدل osi => [
    :: هر لایه در این مدل وضایف خاص خود را دارد و به دیگری مربوط نمیشود
    :: این یک مدل دانشگاهی است و معمولا به صورت عملی مورد استفاده قرار نمیگیرد
    layer-7 : Application (کاربرد) => [
        1) رابط کاربری است و تنها قسمتی از شبکه است که کاربر میتواند با آن در تعامل باشد
        2) برقراری ارتباط با کاربر برای گرفتن / اشتراک گذاری اطلاعات
    ]
    layer-6 : Presentation (نمایش) => [
        1) وظیفه این لایه قالب بندی اطلاعات است
        2) قالب بندی اطلاعات به دو سورت انجام میشود => [
            :: رمز نگاری 
            :: فشرده سازی / ترجمه
        ]
    ]
    layer-5 : Session (نشست) => [
        1) وظیفه این لایه ایجاد کردن و نگهداری کردن . پایان دادن به جلسه ارتباطی است
    ]
    layer-4 : transport (انتقال ) => [
        1) زیاد یا کم بودن میزان اطلاعات ورودی به شبکه را بررسی میکند و در صورت زیاد بودن آن را تکه اکه میکند 
        2)  برای قطعه هایی که تولید کرده است اصطلاحا یک سریال تولید میکند که در مقصد قابل دستسرسی باشد
        3) نوع انتقال ما در این بخش مشخص میشود که آیا سرعت مهم است یا دقت
        // tcp -> دقت | udp -> سرعت
    ]
    layer-3 : network (شبکه) => [
        1) در این لایه آدرس آیپی مقصد و آدرس آیپی مبدا قرار داده میشود
        2) فرایند مسیریابی هم در این آدرس انجام میشود
        3) به آدرس دهی در این لایه آدرس دهی لاجیکال گفته میشود
    ]
    layer-2 : Data Link => [
        1) آدرس دهی فیزیکی انجام میشود
    ]
    layer-1 : Physical => [
        1) اجزای فیزیکی شبکه مثل کابل ها را شامل میشود
    ]
]

:: تشریح مدل TCP / IP => [
    layer-4 -> Application (ترکیب لایه های ۷.۶.۵ مدل قبلی است)
    layer-3 -> transport (مفهموم همین لایه در مدل قبل) 
    layer-2 -> internet (مفهموم مایه ۳ در مدل قبلی)
    layer-1 -> Network Access (ترکیب مدل های ۱و۲ در مدل قبلی)
]

:: تشریح مدل Actual => [
    layer-5 -> Application (ترکیب لایه های ۷.۶.۵ مدل قبلی است)
    layer-4 -> transport (مفهموم همین لایه در مدل قبل)
    layer-3 -> network (مفهموم مایه ۳ در مدل قبلی)
    layer-2 -> Data Link (مفهوم همان مدل قبلی )
    layer-1 -> Physical (مفهوم همان مدل قبلی)
]

:: نام اطلاعات در لایه های مختلف شبکه => [
    :: layer-7 , layer-6 , layer-5 -> Data or Playload 
    :: layer-4 -> Segment 
    :: layer-3 -> Packet
    :: layer-2 -> Frame 
    :: layer-1 -> Bit 
]

:: تفاوت آدرس دهی فیزیکی و منطقی (Logical address VS Physical Address) => [
    :: قبل از هر چیزی یک کامپیوتر برای متصل شدن به یک شبکه نیاز به کارت شبکه دارد
    :: کارت شبکه میتواند به صورت کابلی یا به صورت بیسیم باشد 
    :: Mac (media access control) Address => Physical address => آدرسی است که شرکت سازند بر روی کارت شبکه حک کرده است
    :: IP Address => Logical Address 
    :: آدرس فیزیکی در لایه دوم شبکه کار میکند ولی آدرس منطقی در لایه سوم شبکه کار میکند
    :: آدرس فیزیکی ثابت است (در حالت عادی) اما آدرس منطقی متناسب با شبکه قابل تغییر است
    :: دو مدل آدرس آیپی داریم => IPv4 | IPv6
    :: مک آدرس یک ۴۸ بیتی است که بر مبنای ۱۶ میباشد
    :: مقدار ipv4 برمبنای ۱۰ است و ۳۲ بیتی میباشد
    :: مقدار ipv6 برمبنای ۱۶ است و ۱۲۸ بیتی میباشد
    :: برای اتصال هر کامپیوتر به هر شبکه هم نیاز به آیپی آدرس هست هم نیاز به مک آدرس هست
    :: به تعداد نامحدود میتوان کارت شبکه به یک کامپیوتر وصل کرد هم به صورت بیسیم و هم به صورت کابلی 
    :: به تعداد کارت شبکه های وصل شده به کامپیوتر ما مک آیدی داریم
    :: به ازای هر آدرس مک میتوان به صورت نا محدود آدرس آیپی مرتبط به آن تعریف کرد
    :: OUI -> شناسه خاص 
]

:: ساختار مک آدرس => [
    :: دارای ۴۸ بیت است -> 00-00-00-00-00-00
    :: میتواند شامل اعداد یک تا نه و حروف شa-f شود 
    :: سه جفت اول در این آدرس OUI است و سه جفت بعدی فقط برای دستگاه حال حاضر ما است 
]
